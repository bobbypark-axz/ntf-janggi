## 1. 개요 (Overview)

### 1.1 제품 소개
**Web Janggi**는 별도 설치 없이 웹 브라우저에서 즉시 플레이할 수 있는 HTML5 기반 한국 장기(將棋, Korean Chess) 미니게임이다. 사용자는 컴퓨터(AI)를 상대로 정통 한국 장기 규칙에 따라 대국을 진행하며, 가볍고 빠른 로딩과 직관적인 UI를 통해 짧은 휴식 시간에도 부담 없이 즐길 수 있는 캐주얼 보드 게임 경험을 제공한다.

### 1.2 배경 및 문제 정의
시중의 장기 게임은 대부분 모바일 앱 설치를 요구하거나, 광고/결제 비중이 높고, UI가 복잡하다. 또한 해외 사용자가 접하기 어려운 한자 표기 위주의 인터페이스를 사용하는 경우가 많다. 본 제품은 다음 문제를 해결하는 것을 목표로 한다.

- 설치 부담 없는 즉시 실행(웹 기반)
- 가벼운 단일 페이지 구조로 빠른 로딩 (< 3초)
- 초보자도 직관적으로 이해 가능한 UI/UX
- 단순하지만 도전 가능한 AI 상대

### 1.3 핵심 가치 제안
- **즉시성**: 링크 클릭만으로 플레이 시작
- **친숙성**: 전통 장기판을 충실히 재현
- **접근성**: 데스크탑·모바일 반응형 지원, 초보자용 가이드 제공
- **재미**: 3단계 AI 난이도로 점진적 도전감 제공

## 3. 사용자 정의 (Target Users)

### 3.1 페르소나
**페르소나 A — "주말 장기인" 김민수 (45세)**
- 가끔 장기를 두지만 오프라인 상대가 없음
- 광고가 많은 모바일 앱에 피로감
- 출퇴근길 짧게 한 판 두고 싶어 함

**페르소나 B — "입문자" 박지영 (27세)**
- 장기에 관심은 있으나 규칙이 어려워 시작 못함
- 친구와 함께 배우고 싶지만 보드를 사기는 부담
- 시각적으로 친절한 학습 도구를 원함

### 3.2 사용자 시나리오
1. 사용자가 URL을 입력해 게임에 접속한다.
2. 메인 화면에서 "새 대국 시작"을 선택한다.
3. AI 난이도(쉬움/보통/어려움)를 선택한다.
4. 자신의 진영(초/한)을 선택한다.
5. 차례에 맞춰 기물을 클릭→이동 위치 클릭으로 수를 둔다.
6. AI가 응수하고, 승부가 결정될 때까지 반복한다.
7. 종료 후 결과 화면에서 "다시 하기" 또는 "공유하기"를 선택한다.

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 핵심 기능 (Must-Have, MVP)

**FR-1. 장기판 렌더링**
- 9열 × 10행의 정통 장기판을 캔버스 또는 SVG로 렌더링
- 궁성(九宮) 대각선 표시
- 강(楚河漢界) 중앙 라인 표시
- 반응형(모바일·태블릿·데스크탑)으로 비율 유지

**FR-2. 기물 배치 및 표시**
- 초(楚)·한(漢) 각 16개 기물 정확한 초기 배치
- 기물 종류: 궁(將/帥), 사(士), 차(車), 포(包/砲), 마(馬), 상(象), 졸/병(卒/兵)
- 한자 + 한글 라벨 토글 옵션 지원

**FR-3. 합법 수(Legal Move) 계산**
- 각 기물별 이동 규칙 구현
  - **궁**: 궁성 내 상하좌우 1칸
  - **사**: 궁성 내 대각선/직선 1칸
  - **차**: 직선 무제한 (장애물 통과 불가)
  - **포**: 직선 이동, 반드시 1개의 기물을 넘어야 이동/공격, 포는 포를 넘거나 잡을 수 없음
  - **마**: 한 칸 직진 후 대각선 (멱(蹩) 규칙 적용)
  - **상**: 한 칸 직진 후 대각선 두 칸 (멱 규칙 적용)
  - **졸/병**: 전·좌·우 1칸 (강 건너기 전후 동일, 후진 불가)
- 자신의 궁이 장군 상태가 되는 수는 금지(빅장 포함 규칙 옵션)
- 빅장(맞장군) 금지 규칙 적용

**FR-4. 게임 진행 제어**
- 차례 표시(현재 어느 진영의 차례인지 명확하게)
- 기물 클릭 시 합법 이동 칸 하이라이트
- 잘못된 이동 시도 시 시각/사운드 피드백
- 무르기(되돌리기) 1회 허용 (옵션)
- 기권(항복) 버튼

**FR-5. 승부 판정**
- 외통수(장군 + 회피 불가) 시 즉시 승부 종료
- 빅장(맞장군) 시도 시 시도 측 패배 처리
- 기권 시 즉시 종료
- 무승부 조건: 양측이 합의(옵션) 또는 50수 룰(옵션)

**FR-6. AI 상대**
- 3단계 난이도 제공
  - **쉬움**: 랜덤 + 잡기 우선 (1수 앞)
  - **보통**: Minimax 3~4 ply + Alpha-Beta 가지치기
  - **어려움**: Minimax 5~6 ply + 평가 함수(기물 가치, 위치 가산점)
- 평가 함수 기준 기물 점수: 차 13 / 포 7 / 마 5 / 상 3 / 사 3 / 졸·병 2 / 궁 ∞
- 응답 시간: 보통 난이도 < 2초, 어려움 < 5초 (웹워커로 비동기 처리)

**FR-7. 게임 시작/종료 화면**
- 메인 메뉴: 새 대국, 난이도/진영 선택, 설정
- 결과 화면: 승패 표시, 총 수, 소요 시간, 다시 하기, 메인으로

### 4.2 부가 기능 (Should-Have)

**FR-8. 기보(棋譜) 표시**
- 화면 우측에 양 진영의 수 기록을 순서대로 표시
- 한국식 좌표 표기법(예: 차2진4)

**FR-9. 잡은 기물 표시**
- 양측이 잡은 상대 기물을 보드 옆에 시각화

**FR-10. 사운드 효과**
- 기물 이동, 잡기, 장군, 승부 종료 시 효과음
- 음소거 토글

**FR-11. 설정**
- 한글/한자 라벨 토글
- 사운드 ON/OFF
- AI 응답 시간 표시 ON/OFF
- 무르기 허용 ON/OFF

### 4.3 향후 기능 (Could-Have, 차기 버전)
- 로컬 2인 플레이
- 온라인 멀티플레이
- 게임 저장/불러오기 (LocalStorage 또는 백엔드)
- 튜토리얼 모드 (기물 이동 학습)
- 통계(승률, 평균 수 등) 기록
- 다국어 지원 (영어, 중국어, 일본어)

### 4.4 제외 범위 (Out of Scope, MVP에서 제외)
- 회원가입 / 로그인
- 결제 / 인앱 결제
- 실시간 멀티플레이
- 랭킹 시스템
- 채팅
- 광고

## 5. 사용자 경험 / UI · UX (User Experience)

### 5.1 화면 구성
1. **메인 메뉴 화면**
   - 게임 로고, "새 대국 시작", "설정", "도움말" 버튼
2. **난이도/진영 선택 화면**
   - AI 난이도 3종, 초/한 선택, 시작 버튼
3. **게임 플레이 화면**
   - 중앙: 장기판 (≥ 60% 가로 폭)
   - 우측(또는 모바일 하단): 차례 표시, 기보, 잡은 기물, 컨트롤 버튼(무르기, 기권, 설정)
4. **결과 화면**
   - 승패 결과, 총 수/시간, "다시 하기" / "메인으로"

### 5.2 인터랙션 원칙
- **클릭(또는 탭) → 클릭** 방식: 기물 선택 후 목적지 클릭
- 합법 이동 칸은 옅은 색 도트/링으로 하이라이트
- 잡힌 기물은 페이드아웃 + 효과음
- 장군 발생 시 화면 모서리 적색 점멸 + "장군!" 텍스트 알림

### 5.3 비주얼 디자인 가이드
- 톤앤매너: **전통 + 미니멀** (목재 질감 보드, 깨끗한 라인)
- 색상: 초(파랑/녹청) · 한(빨강)
- 폰트: 본문 — Pretendard, 기물 — Noto Serif KR(한자/한글)
- 다크모드 지원 (옵션)

### 5.4 접근성 (Accessibility)
- 키보드 조작 지원 (방향키 + Enter)
- 시각장애 보조: 색약 모드(파랑↔주황 변환), 명도 대비 WCAG AA 충족
- 모든 인터랙티브 요소 ARIA 라벨 부여

### 5.5 반응형 브레이크포인트
| 디바이스 | 너비 | 레이아웃 |
| --- | --- | --- |
| 데스크탑 | ≥ 1024px | 보드 좌측 + 사이드 패널 우측 |
| 태블릿 | 768~1023px | 보드 상단 + 패널 하단 |
| 모바일 | ≤ 767px | 보드 풀폭 + 패널 토글 시트 |

## 6. 기술 스택 (Tech Stack)

### 6.1 프론트엔드
- **언어**: TypeScript (ES2022)
- **UI 프레임워크**: 바닐라 + Lit (또는 React 18) — 경량성 우선
- **렌더링**: Canvas 2D (성능) 또는 SVG (접근성). MVP는 **Canvas 2D** 권장
- **상태 관리**: 단일 게임 상태 객체 (불변 업데이트)
- **스타일**: CSS Modules 또는 Tailwind (선택)

### 6.2 AI / 로직
- **언어**: TypeScript (게임 엔진과 동일)
- **알고리즘**: Minimax + Alpha-Beta Pruning + 이동순서 정렬(MVV-LVA)
- **실행 환경**: Web Worker (UI 블로킹 방지)

### 6.3 배포 / 인프라
- 정적 호스팅: Cloudflare Pages / Vercel / GitHub Pages
- CDN 캐싱 활용
- 별도 백엔드 없음 (MVP)

### 6.4 분석 / 모니터링
- Plausible 또는 Google Analytics 4 (이벤트 트래킹)

### 6.5 코드 품질
- ESLint + Prettier
- Vitest (단위 테스트, 특히 합법 수 계산기)

## 7. 데이터 모델 (Data Model)

### 7.1 게임 상태 (GameState)
```ts
type Side = 'CHO' | 'HAN'; // 초 / 한
type PieceType = 'GUNG' | 'SA' | 'CHA' | 'PO' | 'MA' | 'SANG' | 'JOL';

interface Piece {
  id: string;
  side: Side;
  type: PieceType;
  position: { x: number; y: number }; // 0~8, 0~9
  alive: boolean;
}

interface Move {
  pieceId: string;
  from: { x: number; y: number };
  to: { x: number; y: number };
  captured?: Piece;
  timestamp: number;
}

interface GameState {
  board: Piece[];
  turn: Side;
  moves: Move[];
  status: 'PLAYING' | 'CHECKMATE' | 'RESIGNED' | 'DRAW';
  winner?: Side;
}
```

### 7.2 영속성
- MVP: 메모리 상태만 유지 (새로고침 시 초기화)
- 차기: LocalStorage에 진행 중인 대국 자동 저장

## 10. 위험 요소 및 대응 (Risks & Mitigation)

| 위험 | 영향도 | 대응 |
| --- | --- | --- |
